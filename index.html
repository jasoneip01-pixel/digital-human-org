0 产品总览

一句话： 数字人组织是一个让企业像管理团队一样编排 AI Agent 的操作系统。填补了从「个人 AI 助手」到「企业级 AI 协作」之间的架构空白。

企业的 AI Agent 部署正在从单个 POC 走向规模化。当前市场只有个人 AI 助手和 AI 开发平台两种产品——中间缺少管理 10-100+ 个 Agent 的岗位定义、权限、协作、审计和成本归因的「组织层」。

🎯 使命

让每个企业员工拥有 10 个数字实习生

🏆 边界

不碰模型 · 不碰 IaaS · 不碰 IM 入口

🔑 抽象

企业组织概念 → 数字人映射层

1 第一性原理推导

从不可推翻的公理开始，推导产品的每一层设计。

公理 A

模型能力已达企业白领的最低胜任线。推理成本年降 50-70%。工具使用（MCP、Function Calling）成熟。

公理 B

多专业 Agent 协作的产出 > 一个全能 Agent。分工减少上下文污染，多 Agent 交叉验证降低幻觉。

公理 C

企业放量的瓶颈不是模型能力，是治理——问责、权限、审计、成本归因缺一不可。

综合： A（可以用）× B（应该用多个）× C（治理是瓶颈）→ 存在显式的市场空白：多 Agent 协作的管理/治理层。

7 Claude Code 模式映射

@agent 标签

Claude Code 用 @agent/tmux 路由任务 → 数字人映射到"@财务Agent"等企业岗位标签

IACP 协议

子 Agent 间 stdin/stdout JSON → 映射为 IACP 协议层的 Request-Response/Pub/Sub/Delegation

Cowork & HIL

Propose→Review→Execute→Verify → 映射为 4 种 HIL 模式：全自动/关键审批/结果审核/仅建议

记忆分层

CLAUDE.md + MEMORY.md + memory/ → 映射为 L1-L4 记忆架构（对话/任务/岗位/组织）
