01 产品定位与愿景

让企业像管理员工一样管理 AI 数字员工

一句话定义：

数字人组织 = 一个企业级的 AI Agent 管理与编排平台。它在个人 AI 助手（如 WorkBuddy）的基础上增加了 「组织」层——支持多 Agent 的岗位定义、权限管理、流程协作、绩效监控和合规审计。

愿景： 成为企业部署 AI 劳动力的「操作系统层」——任何企业都可以像组建一个部门一样，创建、培训、部署、考核一批数字员工。

不是做什么：

  • 不是又一个 AI 对话界面（不是 chat UI）
  • 不是低代码 RPA 平台（不代替 UI 自动化）
  • 不是大模型 API 网关（不止是路由和负载）

是什么： 这三个东西的交集 —— AI Agent 引擎 企业组织管理 工作流编排

02 核心问题

企业在部署 AI Agent 时遇到的系统性矛盾

🔥 问题一：管理黑洞

AI 项目散落在各个部门——市场部用 A 产品、客服部用 B 产品、IT 自建 C——HR 不知道谁负责什么，IT 不知道权限怎么设，老板不知道 ROI 怎么算。

🔥 问题二：治理真空

单个 Agent 好用，但 50 个 Agent 同时运行时：谁对输出负责？数据能不能跨域访问？Agent 的决策怎么留痕？——没有组织层，这些全空着。

🔥 问题三：集成地狱

Agent 需要调 CRM/ERP/OA/企业微信，每个企业 IT 环境不同，一个 Agent 要对接 5-10 个系统。不做集成层，Agent 只是在聊天窗口里回答问题而已。

🔥 问题四：规模失效

1 个 Agent 做 POC 简单，从 1 个到 100 个 Agent 要解决：角色/权限/知识库隔离/成本分摊/性能监控——这在单个 Agent 方案里不存在。

03 用户画像

角色核心诉求决策权重
CTO / CIO统一管理、安全合规、成本可控、不自建60%
HRD数字员工可以「定岗定编」，能考核绩效20%
业务 VP不需要懂 AI 就能用，见效快，不要改流程15%
一线员工数字同事不会抢功、不会甩锅、听指挥5%

关键洞察： CTO/CIO 是核心决策者，他们的核心痛点是「不想再增加一个管理工具」。数字人组织不能成为一个独立的、需要额外运维的 SaaS——它必须长在已有的企业管理框架里，做钉钉/企微/飞书的插件而非替代品。

04 功能架构

四个功能域，按企业部署优先级排列

🔵 域 I：数字员工管理（Digital Workforce Management）

核心——管理数字人的全生命周期

  • 岗位定义器： 创建数字人岗位（岗位名称、职责描述、所需技能、权限范围、KPI 指标）
  • 入职/培训： 绑定知识库 → 测试用例跑通 → 上线
  • 排班调度： 多 Agent 在线/离线/轮班管理
  • 绩效看板： 任务完成率、响应时间、用户满意度、Token 消耗
  • 退役/归档： 下线 Agent，保留审计日志，知识库解绑

🟢 域 II：Agent 推理引擎（Agent Engine）

基础——每个数字人的推理能力

  • 多模型路由： 根据任务复杂度自动选择基础模型（简单查询用 4o-mini，复杂推理用 4o/DeepSeek）
  • Tool Use 框架： Function Calling + 插件市场（API 连接器）
  • 记忆系统： 工作记忆（当前会话）+ 长期记忆（向量库）+ 组织记忆（共享知识库）
  • RAG 引擎： 企业文档检索，支持图谱增强（缓解幻觉）
  • 安全围栏： 敏感词过滤、越权行为检测、人工审核兜底

🟠 域 III：治理合规（Governance & Compliance）

护城河——让 CIO 敢买

  • 权限矩阵： 数字人 × 数据域 × 操作类型（CRUD）的精确控制
  • 审计日志： 每个数字人的每次决策全记录，可回放
  • 租户隔离： 多部门/多子公司的数字人不能跨域访问
  • 成本中心： 每个数字人的 Token 消耗计入对应部门预算

🟣 域 IV：生态系统（Ecosystem）

增长——让伙伴替你解决问题

  • 技能市场： 第三方开发的数字人技能插件（如「SAP 连接器」「CRM 智能客服」）
  • 模板工厂： 预制岗位模板（客服/HR助理/IT运维/销售SDR…）
  • API 网关： 标准化集成接口，支持企业自建系统对接

关键设计决策

决策选择理由
Agent vs 微服务每个数字人 = 一个持久化 Agent 实例企业需要「员工身份」的连续性，不能被负载均衡打散
模型选择多模型路由（非单模型绑定）不同岗位对成本/质量的要求不同，统一绑定浪费
知识库RAG + KG 混合（非纯 RAG）企业关键决策（报销标准/合规条款）需要确定性查询
部署方式SaaS + 私有化双轨中小企业用 SaaS，大型企业/国企必须私有化
工作流DAG 编排 + 人工触发点跨 Agent 流程需要可编程编排；涉及审批的流程必须保留人工节点

06 实现路径

建议分三个阶段，6-12 个月

📦 Phase 1 · MVP（2-3 月）

  • 单个 Agent 的基础能力（对话+Tool Use+RAG）
  • Agent 身份模块（姓名/岗位/权限/话术模板）
  • 1-2 个企业连接器（企微/钉钉）
  • 管理后台基础（创建/监控/日志）
  • 交付指标： 50 家客户 POC，NPS > 30

🏗️ Phase 2 · 组织化（4-6 月）

  • 多 Agent 编排 + 工作流引擎
  • 排班调度 + 绩效考核看板
  • 知识库管理 + 培训环境
  • 审计日志 + 租户隔离
  • 技能插件市场
  • 交付指标： 10 家付费企业，ARPU > ¥5000/月

🌐 Phase 3 · 平台化（6-12 月）

  • 私有化部署方案（K8s Helm Chart）
  • 第三方开发者生态（技能市场 SDK）
  • 模型无关架构（支持国内外所有主流模型）
  • 合规认证（等保/SOC2/数据安全）
  • 交付指标： ARR > ¥5000 万

07 收入模型

三层收入结构，递减依赖推理调用

💰 层 1：订阅（核心收入）

按数字人头数 × 月/年收费。分层定价（基础-专业-企业）。

预期占比：70%

📱 层 2：增值（盈利杠杆）

预置技能模板、API 调用配额包、模型路由升舱、专属知识库空间

预期占比：20%

🔧 层 3：服务（获客锁客）

实施部署（¥30-200万/单）、定制开发、培训、私有化运维年费

预期占比：10%

定价原则：

  • 不按 Token 收费（企业厌恶变量费用的不确定性 —— CFO 要的是「下个月固定花多少」）
  • 不按对话次数收费（同理，变量不可控）
  • 按人头固定月费，上限不限量（企业的直觉是「买的越多越值」）
  • 模型推理成本随规模下降，但定价不掉 —— 盈利空间靠规模扩大和技术优化自然增厚

10 竞品定位

竞品定位与我们不重叠的地方
WorkBuddy
腾讯		 个人桌面 AI Agent 个人效率工具 vs 企业组织管理
我们面向 CIO/HRD，WorkBuddy 面向个人用户
Coze / Dify AI 应用开发平台 AI 应用搭建工具 vs AI 员工管理平台
企业用 Coze 搭一个 Chatbot，我们搭一个「部门」
钉钉/企微 AI 平台内嵌 AI 助手 它们是渠道/入口，我们是管理后台
互补而非竞争——数字人组织应该对接它们
Glean / Atomicwork 企业知识搜索+AI 知识搜索 vs AI 劳动力管理
Glean 帮你找到信息，数字人组织帮你「做了这件事」

竞争壁垒（按重要性排序）：

  1. 组织抽象层： 把「岗位/角色/权限/流程」映射到 AI Agent 系统的能力——这是产品设计最难复制的东西
  2. 企业集成深度： 对接 20+ 企业系统的连接器网络，时间积累的壁垒
  3. 合规框架： 审计/权限/数据隔离的完整性和合规背书
  4. 模型中立： 不绑定任何单一模型，企业在国内外模型间自由切换，规避供应链风险

11 GTM 策略

🎯 Phase 1：场景切入（0-3 月）

  • 选 2 个高频、低风险的场景做 MVP：客服+IT 运维
  • 免费 POC 给 50 家目标客户，换反馈+案例
  • 不追逐大客户——先在小客户跑闭环

🚀 Phase 2：行业标杆（3-9 月）

  • 找到 2-3 个行业标杆客户（电商/金融/科技优先）
  • 打造可复制的行业模板
  • 渠道合作：与企微/钉钉服务商联合销售
  • 定价从「项目制」切换到「SaaS 订阅」

🌱 Phase 3：网络效应（9-18 月）

  • 技能市场开放第三方开发者入驻
  • 模板广场——用户上传和复用岗位模板
  • API 开放——让 ISV 在平台上嵌入数字人
  • 代理商网络（按销售额分成 20-30%）

12 关键风险与应对

风险概率应对
大厂入场（腾讯/阿里做「数字人组织」） 高 差异化定位：不做平台，做「管理AI员工的操作系统」。大厂做 Agent，我们做管理 Agent 的框架。同时做深系统集成——钉钉不会帮客户对接飞书，但我们会。
企业客户不信任 AI 决策 中高 产品设计原则：数字人只能做「参数可配、结果可预期」的事。关键决策走人工审批。审计日志全留痕。给 CIO 一个「随时可以关掉」的按钮。
模型成本下降 → 竞品降价 低 定价与模型成本脱钩，锚定的是「替代人力成本」。企业招一个员工的月成本是 ¥8,000-15,000，数字人收 ¥699-1,999，有 5-10 倍替代价值空间。
召回率瓶颈 中 不依赖纯 RAG，加入知识图谱层做确定性查询。同时划定「数字人能做 vs 不能做的事」的边界，超出边界升级到真人，不在模糊区硬做。
数据安全合规（等保/个保法） 中 私有化部署从 Phase 1 就开始规划架构，不做事后补救。为大型客户提供「数据不出域」选项。
