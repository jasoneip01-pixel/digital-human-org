01 产品定位与愿景

让企业像管理员工一样管理 AI 数字员工

一句话定义： 数字人组织 = 一个企业级的 AI Agent 管理与编排平台。它在个人 AI 助手（如 WorkBuddy）的基础上增加了 「组织」层——支持多 Agent 的岗位定义、权限管理、流程协作、绩效监控和合规审计。

愿景： 成为企业部署 AI 劳动力的「操作系统层」——任何企业都可以像组建一个部门一样，创建、培训、部署、考核一批数字员工。

不是做什么

  • 不是又一个 AI 对话界面
  • 不是低代码 RPA 平台
  • 不是大模型 API 网关
AI Agent 引擎 企业组织管理 工作流编排

02 核心问题

企业在部署 AI Agent 时遇到的系统性矛盾

🔥 管理黑洞

AI 项目散落在各部门——HR 不知谁负责，IT 不知权限怎么设，老板不知 ROI 怎么算。

🔥 治理真空

50 个 Agent 同时运行时：谁对输出负责？数据能不能跨域？决策怎么留痕？

🔥 集成地狱

Agent 要调 CRM/ERP/OA/企微，不做集成层，Agent 只在聊天框里回答问题。

🔥 规模失效

1 个 Agent POC 简单，1→100 个要解决角色/权限/知识库隔离/成本分摊/性能监控。

03 用户画像

角色核心诉求决策权重
CTO / CIO统一管理、安全合规、成本可控、不自建60%
HRD数字员工可以定岗定编，能考核绩效20%
业务 VP不需要懂 AI 就能用，见效快15%
一线员工数字同事不会抢功、不会甩锅5%

关键洞察： CTO/CIO 是核心决策者。数字人组织必须长在已有企业管理框架里，做钉钉/企微/飞书的插件而非替代品。

05 系统架构

四层架构，从 Agent 内核到企业管理层

🟡 管理层

Workspace · 配额管理 · 计费 · 审计日志 · 租户隔离 · 成本报表

🟠 组织层

岗位定义 → 数字人编排 → 工作流引擎 → 绩效监控 → 培训环境 · 跨 Agent 协作 · 排班调度 · 角色隔离

🟢 数字人层

身份模块 · 推理引擎 · 技能/Tool Use · 记忆持久化 · 多模型路由 · RAG 引擎 · 安全围栏 · 对话管理

🟣 集成层

企微/钉钉/飞书 · SAP/OA/CRM/ERP · 数据库 · IM 消息通道 · API 网关 · Webhook · 企业 SSO · 数据管道

关键设计决策

决策选择理由
Agent vs 微服务持久化 Agent 实例企业需要「员工身份」的连续性
模型选择多模型路由不同岗位对成本/质量要求不同
知识库RAG + KG 混合关键决策需要确定性查询
部署方式SaaS + 私有化双轨中小用 SaaS，大型必须私有化
工作流DAG 编排 + 人工触发点审批流程必须保留人工节点

08 定价体系

版本定价核心能力目标客户
基础版¥199/人/月基础 Agent · 限 5 数字人 · 1000 次/月推理小型团队
专业版¥699/人/月完整 Agent + 工作流 · 不限量 · 审计 · 技能市场中型企业
企业版¥1,999/人/月全部 + 私有化部署 · 专属微调 · SLA 99.9%大型企业/国企

定价原则： 不按 Token 收费——企业厌恶变量费用。按人头固定月费，上限不限量。模型成本随规模下降，盈利空间自然增厚。

10 竞品定位

竞品定位差异
WorkBuddy个人桌面 AI Agent个人效率 vs 企业组织管理
Coze / DifyAI 应用开发平台搭 Chatbot vs 搭一个「部门」
钉钉/企微 AI平台内嵌 AI 助手渠道/入口，我们是管理后台
Glean / Atomicwork企业知识搜索+AI找到信息 vs 做了这件事

竞争壁垒： 1. 组织抽象层（岗位/角色/权限→Agent） 2. 企业集成深度 3. 合规框架 4. 模型中立

11 GTM 策略

🎯 Phase 1 · 场景切入

0-3 月。选 2 个场景做 MVP（客服+IT运维），50 家客户免费 POC。

🚀 Phase 2 · 行业标杆

3-9 月。2-3 个标杆客户，打造可复制模板，渠道合作联合销售。

🌱 Phase 3 · 网络效应

9-18 月。技能市场开放第三方开发者，API 开放，代理商网络。

12 关键风险与应对

风险概率应对
大厂入场高差异化定位：不做平台，做管理AI的操作系统
客户不信任 AI中高关键决策人工审批，审计日志全留痕
模型成本下降低定价锚定「替代人力成本」，5-10 倍替代空间
召回率瓶颈中RAG+KG 混合，超出边界升级到真人
数据安全合规中私有化架构从 Phase 1 开始规划
