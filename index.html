0 执行摘要

一句话： 数字人组织是一个让企业像管理团队一样编排 AI Agent 的操作系统——填补了从「个人 AI 助手」到「企业级 AI 协作」之间的架构空白。

企业 AI 部署正在经历从 POC 到规模化的临界点。但当前市场只有两类产品：个人级 AI 助手（Copilot/WorkBuddy）和 AI 开发平台（Coze/Dify）。中间层缺失——没有产品能管理 10-100+ 个 Agent 的岗位定义、权限、协作、审计和成本。

🎯 目标市场

中国 2000 万家企业中的 AI 先行者。首期目标：1000 家付费客户，10 万数字人头

🏆 核心优势

组织抽象层（岗位/角色/权限→Agent）+ @agent 标签路由 + MCP 生态 + 模型中立

💰 收入目标

Year 3 ARR ¥1.2 亿 · 单位利润率 58%（企业版）· 客户 LTV/CAC > 8×

1 核心问题——跨越 POC 死亡谷

企业 AI 从试点到大规模部署的四个系统性断裂

🔥 管理黑洞

AI 项目散落各部门——HR 不知谁负责，IT 不知权限怎么设，老板不知 ROI 怎么算。缺少统一管理面。

🔥 治理真空

50 个 Agent 同时运行时：谁对输出负责？数据能不能跨域？决策怎么留痕？AI 的问责链在传统框架中不存在。

🔥 集成地狱

Agent 要调 CRM/ERP/OA/企微。不做集成层，Agent 只在聊天框里回答问题——集成占大模型项目 60-70% 的工作量。

🔥 规模失效

1 个 Agent POC 简单，1→100 个要解决角色/权限/知识库隔离/成本分摊/性能监控——这些基础设施不存在。

2 方案——企业 AI 协作操作系统

从「数字员工管理平台」进化到「企业 AI 协作操作系统」——三层核心价值

📋 层 1 · 管理

数字员工全生命周期：岗位定义→入职培训→排班→绩效→退休。CTO 看到一个团队，不是一堆 API。

🤝 层 2 · 协作

@agent 标签路由 + IACP 协议：Agent 之间、Agent 与人类之间的标准化协作。每次交互可追溯、可审计。

🔒 层 3 · 治理

Prompt 注入防护 · 权限矩阵 RBAC/ABAC · 不可变审计链 · 跨层安全基础。通过企业安全审计。

关键差异： 不是又一个 AI 对话界面，不是低代码 RPA 平台，不是大模型 API 网关。是管理 AI 劳动力的操作系统层。

5 竞争格局与差异 深度

我们不是做一个更好的 Copilot——我们在创建一个新品类。

竞品定位数字人组织的差异威胁
Microsoft Copilot个人生产力 AI组织层缺失 → 无法管理 100 个 AI中
WorkBuddy个人桌面 AI Agent个人效率 vs 企业组织管理低
Coze / DifyAI 应用开发平台搭 Chatbot vs 搭一个「部门」中
钉钉/企微 AI平台内嵌助手渠道/入口，我们是管理后台高
Glean / Atomicwork企业知识搜索+AI找到信息 vs 把事情做完低
Salesforce AgentforceCRM 内置 Agent绑定 Salesforce vs 模型中立中

🛡️ 护城河 1 · 组织抽象层

岗位/角色/权限→Agent 的映射是独创的。竞品做的是 Agent 编排，我们做的是 Agent 人力资源管理。

🛡️ 护城河 2 · 企业集成深度

长在钉钉/企微/飞书生态里的管理后台，不是独立 SaaS。减少用户迁移成本。

🛡️ 护城河 3 · IACP 协议

Agent 间通信标准化。当企业有 100 个 Agent 时，没有 IACP 就是 100 个孤岛。

🛡️ 护城河 4 · 模型中立

不绑定任何模型。企业可以用 DeepSeek 做成本敏感任务，用 Claude 做推理密集型任务。
