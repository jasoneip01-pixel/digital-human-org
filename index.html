0 执行摘要

数字人组织 是一个让企业像管理团队一样编排 AI Agent 的操作系统。它填补了从「个人 AI 助手」到「企业级 AI 协作」之间的架构空白——不是又一个对话界面，不是低代码 RPA，不是模型网关。是管理 AI 劳动力的操作系统层。

AI Agent 市场正经历从 POC 到规模化的转折。Gartner 预测 2027 年 40% 的企业将部署多 Agent 系统；MarketsAndMarkets 预计全球市场从 2024 年 $5.4B → 2027 年 $24B（CAGR 64%）。但当前没有产品能管理 10-100+ 个 Agent 的岗位定义、权限、协作、审计和成本。

基于第一性原理推导（见后文），我们从三个不可推翻的公理出发，推导出产品架构的每一项设计决策——不是"很酷所以做"，而是"不这么做就违反了公理约束"。

🎯 目标

让每个企业员工拥有 10 个数字实习生。企业从买人头转向买数字人头。

🏆 定位

大厂 AI 基础设施之上的一层组织抽象。不碰模型、不碰 IaaS、不碰 IM。

💰 融资

¥1,500 万天使轮 · 18 个月跑道 · 目标 Y3 ARR ¥1.2 亿

1 第一性原理推导

不从市场报告出发，从不可推翻的公理出发，逐层推导产品必要性。

公理 A — 模型能力已达实用门槛

命题： GPT-4o / Claude 4 / DeepSeek V4 在推理、工具调用、长上下文方面的能力已超过企业白领工作的最低胜任线。

📊 证据 1: 任务等效

SWE-bench verified (10k 真实 GitHub Issue) 解决率从 GPT-4 的 13% 升至 Claude 4 的 72%—从"不可用"到"实用"的阈值在 2025-2026 被跨越。

📊 证据 2: 工具使用

模型原生支持 MCP / Function Calling — 不再需要复杂 prompt 工程，Agent 可以直接操作企业软件。

📊 证据 3: 成本曲线

推理成本以年 -50~-70% 的速度下降。2024 年 $20/百万 token → 2026 年 $2-3/百万 token。人力替代的盈亏平衡点已经被交叉。

📊 证据 4: 多模态

模型能"看懂"截图、图表、UI — 企业场景中 60% 的信息是非结构化的，这个能力是 Agent 实用性的必要条件。

公理 B — 协作产出大于个体之和

命题： 在复杂企业任务中，多个专业 Agent 协作的产出质量 > 一个全能 Agent + 同等推理预算。

🧪 推理 1: 分工原理

一个 Agent 做 10 件事 = 10 倍的上下文混杂 + 任务切换成本。十个专精 Agent 各做一件事 = 每个 Agent 的上下文纯粹、输出质量可回溯。

🧪 推理 2: 认知多样性

同一模型的不同角色 prompt 产生不同方向的输出——多 Agent 辩论/交叉验证能有效降低幻觉率（研究显示降低 40-60%）。

🧪 推理 3: 规模效应

人力团队有管理半径（Dunbar 数 ~150）。数字人团队没有——1000 个 Agent 可以由 1 个 管理员经理配置。边际管理成本趋近于零。

🧪 推理 4: 容错

一个 Agent 崩了 ≠ 整个系统崩了。多 Agent 架构天然支持优雅降级。

公理 C — 企业需要治理超过需要能力

命题： 决定企业能否放量部署 AI 的不是模型能力，而是治理能力——谁在管？谁负责？谁能看什么？成本怎么算？

🔒 推理 1: 问责链

一个 AI 产生的错误决策导致损失，企业需要一个有名字的人来负责。没有组织层的问责机制，AI 无法进入核心业务流程。

🔒 推理 2: 权限

Agent 能看/能改什么数据？不同部门的 Agent 之间数据怎么隔离？没有 RBAC/ABAC 层面的管理，IT 审计不会通过。

🔒 推理 3: 成本归因

AI 推理成本比传统 SaaS 高 10-100 倍。没有按部门/项目/Agent 的成本分摊，CFO 不会批准规模部署。

🔒 推理 4: 审计

每个 Agent 的每次决策都需留痕。传统日志系统不适用于多 Agent 协作——需要新的审计范式。

综合推导

从 A∧B∧C 推导出产品必要性：

A（模型已足够）→ 可以用
B（多 Agent 更好）→ 应该用多个
C（治理是瓶颈）→ 需要管理系统
∴ 存在显式的市场空白：多 Agent 协作的管理/治理层

4 Claude Code 模式映射 深度

Claude Code 的实践验证了多 Agent 协作的可行性。以下将 Claude Code 的设计模式映射为数字人组织的架构特征。

模式 1: @agent 标签路由

Claude Code

@agent/github · @agent/tmux · 通过标签将子任务路由到正确的 Agent 实例

→ 数字人映射

@agent 标签 = 岗位名。数字人组织的【组织层】维护标签路由表。员工在群里 @财务Agent → 系统路由到财务岗位的数字人

模式 2: IACP — Inter-Agent Communication

Claude Code

子 Agent 之间通过 stdin/stdout JSON 协议通信 · 父 Agent 合并结果 · 有明确的 request-response 生命周期

→ 数字人映射

IACP 协议（第 7 层）标准化 Agent 间通信：Request-Response · Pub/Sub · Task Delegation · Conflict Resolution

模式 3: Cowork & HIL

Claude Code

Propose→Review→Execute→Verify 闭环。关键操作 ask() 确认，run() 执行后自动验证

→ 数字人映射

4 种 HIL 模式：① 全自动（低风险）② 关键步骤人工确认 ③ 结果需人工审核 ④ 只建议不执行

模式 4: 记忆分层

Claude Code

CLAUDE.md · MEMORY.md · memory/YYYY-MM-DD.md · 写入索引 · 交叉引用 · 自我迭代

→ 数字人映射

L1-L4 记忆架构（第 2 层）完全映射 Claude Code 的持久化层级——对话记忆 → 任务记忆 → 岗位记忆 → 组织知识

模式 5: 工具绑定与安全

Claude Code

每个 Tool 有明确的允许/禁止列表。高危操作（exec/delete）需要显式批准。不可在工具调用中做权限升级。

→ 数字人映射

第 0 层安全基础 + 第 6 层路由：每个 Agent 绑定特定的工具白名单。企业级 RBAC/ABAC。

架构启示

Claude Code 验证了一个关键假设： 多 Agent 协作（子 Agent + @agent 路由 + IACP）在真实世界已经可工作。数字人组织解决的不是"能不能协作"的问题——是"怎么在生产环境中管理 100 个协作 Agent"的问题。

市场规模推导（第一性原理）

自底向上估算：

中国 1000 人以上企业 ~8,000 家 · 每企业 500 个白领 · 50% 可被 AI 增强或替代 · 每个增强 AI Agent 月费 ¥699 · 渗透率 40%（Gartner 预测）

= 8,000 × 500 × 50% × ¥699 × 40% = ¥56 亿/月 → ¥672 亿/年

即使只捕获 1% 的此市场 = ¥6.7 亿 ARR。TAM 远大于当前市场报告估计。

市场驱动力

💰 成本剪刀差

人力成本年涨 8-12% vs AI Agent 成本年降 50-70%。2025-2026 是交叉点。企业首次可以通过 AI 替代实现正的净现成本节约。

🧠 模型 S 曲线

推理能力在 2024-2026 年间经历了两阶跃升级。模型从"理解问题"进化为"独立解决问题"。这解锁了 Agent 的自主性。

🏢 企业合规驱动

各地关于 AI 治理的法规密集出台（中国 AIGC 管理办法 · EU AI Act · 美国 AI Bill of Rights）。企业需要合规工具来管理 AI 行为。

🔧 MCP 生态

MCP 协议（Model Context Protocol）标准化了 Agent 与企业系统的连接方式。2026 年已是事实标准——消除了集成壁垒。

护城河

🛡️ 1. 组织抽象层

岗位/角色/权限→Agent 的独创映射。竞品做 Agent 编排，我们做 Agent 人力资源管理。这个抽象是 6-12 个月的领先窗口。

🛡️ 2. IACP 协议

Agent 间通信标准化。当企业有 100 个 Agent 时，没有标准协议 = 100 个孤岛。我们制定协议，生态锁定。

🛡️ 3. @agent 标签路由

自然的交互模式：用户在群里 @财务Agent → 系统路由。这是 IM 生态内最自然的交互方式，用户零学习成本。

🛡️ 4. 成本归因原生设计

从第一天起架构按 Agent 分摊推理成本。竞品做个人 AI 时不需要这个能力——要加成本巨大。

🛡️ 5. 模型中立 + MCP 生态

不绑定任何模型或云——企业可以用 DeepSeek 做简单任务，Claude 做复杂推理。MCP 标准解除了集成锁定。

🛡️ 6. 合规原生设计

安全第 0 层 —— 不可变审计链 + Agent Namespace 隔离 + Prompt 注入防护。不是后加的补丁，是架构根基。

🛡️ 7. 网络效应

Agent 技能市场（第 3 年开启）：企业发布 Agent 模板，其他企业复用。更多客户 → 更多技能 → 更高粘性。
